**المشكلة:** `Permission denied to myrahmd85-ship-it`

1. **حمّل GitHub Desktop:**
   - اذهب إلى: https://desktop.github.com
   - حمّل وثبّت البرنامج

2. **افتح المشروع في GitHub Desktop:**
   - افتح GitHub Desktop
   - اضغط: File → Add Local Repository
   - اختر المجلد: `D:\عمار  git init`
   - اضغط Add Repository

3. **سجّل دخولك:**
   - اذهب إلى: File → Options → Accounts
   - سجّل دخول بحساب GitHub الصحيح: `omier-365`

4. **انشر المشروع:**
   - اضغط "Publish repository"
   - اسم Repository: `amar-cosmatics`
   - ✅ تأكد من إلغاء اختيار "Keep this code private" (إذا كنت تريده عام)
   - اضغط "Publish Repository"

**أو الحل البديل (Terminal):**

```bash

git remote remove origin

# 2. أنشئ Personal Access Token:
# اذهب إلى: https://github.com/settings/tokens
# اضغط "Generate new token (classic)"
# اختر: repo (جميع الصلاحيات)
# انسخ الـ Token

# 3. أضف الـ origin مع الـ Token
```

## الخطوة 2: النشر على Vercel 🚀

### بعد رفع المشروع على GitHub:

1. **اذهب إلى Vercel:**
   - افتح: https://vercel.com/new
   - سجّل دخول بحساب GitHub

2. **استورد المشروع:**
   - اضغط "Import Git Repository"
   - اختر: `omier-365/amar-cosmatics`
   - اضغط "Import"

3. **أضف Environment Variables:**
   
   في قسم "Environment Variables":
   
   ```
   Name: NEXT_PUBLIC_SUPABASE_URL
   Value: [سنضيفها لاحقاً من Supabase]
   
   Name: NEXT_PUBLIC_SUPABASE_ANON_KEY
   Value: [سنضيفها لاحقاً من Supabase]
   
   Name: NEXT_PUBLIC_WHATSAPP_NUMBER
   Value: +201202119847
   ```

4. **انشر:**
   - اضغط "Deploy"
   - انتظر 2-3 دقائق
   - ستحصل على رابط مثل: `https://amar-cosmatics.vercel.app`

## الخطوة 3: إعداد Supabase (اختياري - للبيانات الحقيقية) 💾

### إذا كنت تريد استخدام قاعدة بيانات حقيقية:

1. **أنشئ حساب:**
   - اذهب إلى: https://supabase.com
   - سجّل دخول وأنشئ مشروع جديد

2. **أنشئ الجداول:**
   - اذهب إلى SQL Editor
   - افتح ملف `supabase-schema.sql` من مشروعك
   - انسخ محتوياته والصقها في SQL Editor
   - اضغط "Run"

3. **احصل على المفاتيح:**
   - اذهب إلى: Settings → API
   - انسخ:
     - **Project URL** (هذا هو NEXT_PUBLIC_SUPABASE_URL)
     - **anon public key** (هذا هو NEXT_PUBLIC_SUPABASE_ANON_KEY)

4. **حدّث المتغيرات في Vercel:**
   - ارجع إلى Vercel Dashboard
   - اذهب إلى: Settings → Environment Variables
   - حدّث القيمتين
   - اذهب إلى: Deployments → اضغط ⋯ على آخر deployment → "Redeploy"
